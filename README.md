# Writing Bot RAG

Writing Bot RAG is a Codex skill suite for evidence-aware writing, claim-evidence auditing, citation-risk review, and reviewer-response drafting. It consumes local RAG evidence-pack workspaces and Reviewer Hell critique packs generated by Research Bot RAG.

The project is designed for academic and technical writing workflows where citation boundaries, manuscript claims, reviewer replies, and artifact-backed revisions must be kept explicit.

## What This Repository Contains

- `skills/writing-bot`: the primary Writing Bot entry skill.
- `skills/rebuttal-shield`: a thin alias skill for defending against Reviewer Hell critique packs.
- `src/research_rag`: shared local RAG workspace and Reviewer Hell runtime code used by the Writing Bot wrappers.
- `samples`: tiny synthetic connector-style samples for smoke testing.
- `tests`: lightweight Reviewer Hell behavior tests.

This repository intentionally does not contain private corpora, PDFs, Gmail exports, Drive exports, vector indexes, or model weights.

## Core Workflows

Audit a claim against an existing evidence-pack workspace:

```powershell
python "skills/writing-bot/scripts/rag_writing_audit.py" `
  --workspace "<workspace path or latest>" `
  --claim "<claim to audit>" `
  --root "<RAG_ROOT>"
```

Attach a researcher memo or artifact to a Reviewer Hell critique:

```powershell
python "skills/writing-bot/scripts/rag_attach_artifact.py" `
  --workspace "<workspace path or latest>" `
  --critique-id C001 `
  --file "<artifact path>" `
  --artifact-type simulation_result `
  --researcher-memo "<what this artifact shows, does not show, and what claim boundary it supports>" `
  --root "<RAG_ROOT>"
```

Optional metric sanity fields:

```powershell
--metric settling_time --metric-value 0.05 --unit s --baseline-value 0.08 --threshold 0.1 --threshold-source advisor --threshold-direction le
```

Draft a defense and advisor shield:

```powershell
python "skills/writing-bot/scripts/rag_student_defense.py" `
  --workspace "<workspace path or latest>" `
  --root "<RAG_ROOT>"

python "skills/writing-bot/scripts/rag_advisor_shield.py" `
  --workspace "<workspace path or latest>" `
  --root "<RAG_ROOT>"
```

## Minimal Gate Policy

Writing Bot RAG is researcher-led. It does not certify claims as true. It keeps evidence boundaries visible and uses three states:

- `usable_for_draft`
- `needs_human_judgment`
- `blocked`

Blocked items go to researcher action items, not final rebuttal prose. Metric sanity flags do not prove claims; they only reveal missing units, baseline ambiguity, threshold ambiguity, or missing threshold sources.

## Installation

```powershell
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install -e .
```

## Local RAG Root

The runtime resolves the RAG root in this order:

1. `RESEARCH_RAG_ROOT`
2. `<RAG_ROOT>` if it exists
3. `~/.research_rag`

For public use, set `RESEARCH_RAG_ROOT` explicitly and keep private corpora outside the Git repository.
